---
description: Git add, commit, pushを実行する
---


以下の手順でgitコミットを実行してください：

1. **変更の確認**
   - `git status` を実行して、コミットする変更があるか確認する
   - `git diff` を実行して、変更内容を確認する


2. **ステージングされていない変更の追加**
   - 変更されたファイルを `git add` でステージングエリアに追加する
   - 新規ファイルも含めて適切にステージングする


3. **コミットメッセージの作成**
   - 変更内容を分析して、適切なコミットメッセージを作成する
   - コミットメッセージは以下の形式に従う：
     - 1行目：変更の概要（50文字以内）
     - 2行目：空行
     - 3行目以降：詳細な説明（必要に応じて）
   - 日本語または英語で記述する（プロジェクトの言語に合わせる）


4. **コミットの実行**
   - `git commit` を実行してコミットを作成する
   - コミット後、`git log -1` で作成されたコミットを確認する

5. **プッシュの実行**
   - commitされた内容で現在のブランチからリモートのブランチに`git push origin`でプッシュする


**注意事項：**
- .env、credentials.json など、機密情報を含むファイルはコミットしない
- そのようなファイルが検出された場合は、ユーザーに警告する
- node_modules や .next などのビルド成果物は通常コミットしない（.gitignoreで除外されているべき）
- 必ずステップごとにコマンドを実行して良いか確認をすること
